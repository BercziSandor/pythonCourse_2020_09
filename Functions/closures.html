Definíció

CLOSURE, LEXICAL CLOSURE: Olyan függvény, amin belül definiálunk egy másik függvényt; a külsõ (factory) függvény a belsõt adja vissza, amely megõrzi a keletkezésekor fennálló belsõ állapotot (hívási paraméterek és lokális változók értékét).

(Igen, jól érzékeli az olvasó: ez így egyszerre hat tudományoskodónak, érthetetlennek és feleslegesnek. Valójában egyik sem.)

Ismerkedésként nézzünk meg egy nagyon egyszerû closure-t: 

def outer_func(outerParam):
    def inner_func(innerParam):
        print('inner_func says: outerParam was',outerParam)
        print('inner_func says: local_var was:',local_var)
        print('inner_func says: innerParam is:',innerParam)

    local_var = '*LOCAL VARIABLE*'

    return inner_func    # returns the inner function object

f = outer_func('OUTER')  # a new function (inner_func) is created
print(f)
f('inner func parameter')


# <function outer_func.<locals>.inner_func at 0x0232CB70>
# inner_func says: outerParam was 'OUTER'
# inner_func says: local_var was: *LOCAL VARIABLE*
# inner_func says: innerParam is: inner func parameter
A külsõ függvény csak gyárként (factory) szolgál, amikor felhívjuk, akkor visszaadja a belsõ függvényt. Az új, f függvény tulajdonképpen a belsõ függvényre mutató referencia; amikor õt felhívjuk, inner_func fog lefutni. És amikor lefut, láthatjuk, hogy megjegyezte a külsõ függvénynek azt az állapotát (lokális változóit és bemenõ paramétereit), ami az õ elkészítésekor érvényes volt.

...és ugyanez objektummal

Nagyon tanulságos megnézni egy másik módszert a closure által megjegyzett információk tárolására. Ha valamilyen adatot meg kell jegyeznie a programnak, akkor az elsõ gondolatunk nyilván egy "hagyományos" objektum lesz (és ezentúl a második gondolatunk egy closure függvény, amint megszoktuk ezt az új fogalmat). 

class ClosureClass:
    def __init__(self,outerParam):
        self.outer_param = outerParam
        self.local_var = '*LOCAL VARIABLE*'

    def _inner_func(self,innerParam):
        print('inner_func says: outerParam was',self.outer_param)
        print('inner_func says: local_var was:',self.local_var)
        print('inner_func says: innerParam is:',innerParam)

clos_obj = ClosureClass('OUTER')
f = clos_obj._inner_func
f('inner func parameter')


# inner_func says: outerParam was 'OUTER'
# inner_func says: local_var was: *LOCAL VARIABLE*
# inner_func says: innerParam is: inner func parameter
Azt hiszem, jól látszik, mennyivel több segéd-kódot kellett írnunk a két sztring megjegyzéséhez – ez remélhetõleg felkelti a szimpátiát a closure irányában.

...vagy hívható objektummal

Meg lehet oldani a feladatot egy kicsit direktebb, egyszerûbb módon is, ha a segédobjektumunkat hívhatóvá tesszük (tehát ilyen módon gyakorlatilag egy függvényt állítunk elõ). 

class CallableClosureClass:
    def __init__(self,outerParam):
        self.outer_param = outerParam
        self.local_var = '*LOCAL VARIABLE*'

    def __call__(self,innerParam):
        print('inner_func says: outerParam was',self.outer_param)
        print('inner_func says: local_var was:',self.local_var)
        print('inner_func says: innerParam is:',innerParam)

f = CallableClosureClass('OUTER')  # __init__
f('inner func parameter')          # __call__


# inner_func says: outerParam was OUTER
# inner_func says: local_var was: *LOCAL VARIABLE*
# inner_func says: innerParam is: inner func parameter
A "hagyományos" objektummal való megoldás akkor lehet elõnyösebb (olvashatóbb) a closure függvény-objektumnál, amikor többféle funkcionalitásra, azaz nem csak egyetlen metódusra van szükségünk.
A closure függvénynél a bemenõ pataméterek játsszák azt a szerepet, mint az objektumoknál a konstruktor paraméterei.

Alkalmazások

Default paraméterek beállítása

A closure-t használhatjuk arra, hogy egy sokparaméteres függvény valahány paraméterének default értéket adjunk, ezáltal olvashatóbbá téve a programunkat. Legyen pl. egy négy paraméteres függvényünk, amelyet egy programrészben sokszor használunk úgy, hogy két paraméter értéke mindig 1000, ill. 100: 

def f_many_pars(a,b,c,d):
    return a + b + c + d
          ⇓
def reduce_pars(a,b):
    def f_inner(c,d):
        return f_many_pars(a,b,c,d)

    return f_inner

f_less_pars = reduce_pars(1000,100)
x = f_less_pars(10,1)
print(x)
x = f_less_pars(20,2)
print(x)

Egyetlen metódusú osztályok kiváltása

Tegyük fel, hogy egy programban számos helyen használnunk kell egy videotömörítõ függvényt, amely különbözõ videókat tömörít, de állandó paraméterekkel (felbontás, formátum...). Ezeket a paramétereket csak egyszer állítjuk be, de sokszor hívjuk a függvényünket. A paraméterek megjegyzésére a legcsúnyább megoldás globális változók alkalmazása; ehelyett jobb, ha egy segdosztályt definiálunk, de ennél is tömörebb és elegánsabb closure alkalmazása. 

                                    osztály            ⇒            függvény
class CompressClass:                                                  def outer_compress(resolution, format):
    def __init__(self, resolution, format):                               def inner_compress(inputVideo):
        self._resolution = resolution                                          # ... uses resolution, format
        self._format = format                                             return inner_compress

    def compress(inputVideo):
        # ... uses self._resolution, self._format

comp_obj_1 = CompressClass('1080p','mp4')                              comp_func_1 = outer_compress('1080p','mp4')
comp_obj_1.compress(input_video)                                       comp_func_1(input_video)
comp_obj_2 = CompressClass('1080p','mp3')                              comp_func_2 = outer_compress('1080p','mp3')
comp_obj_2.compress(input_video)                                       comp_func_2(input_video)

Kifelé rejtett attribútumok

A gyártó függvény paraméterei és lokális változói láthatóak a legyártott függvény számára, de kívülrõl ezek nem hozzáférhetõek. 

def func_factory(startParam):
    def func():
        print('I am func, my start parameter:',startParam)

    return func

f1 = func_factory(10)
f2 = func_factory(999)

f1()
f2()


# I am func, my start parameter: 10
# I am func, my start parameter: 999
Ha függvényattribútumot használnánk, az kívülrõl is változtatható lenne: 
def func_factory():
    def func():
        print('I am func, my start parameter:',func_factory.startParam)

    return func

f1 = func_factory()
func_factory.startParam = 10
f1()
func_factory.startParam = 999
f1()


# I am func, my start parameter: 10
# I am func, my start parameter: 999
Az elsõ megoldásnál el tudjuk rejteni a startParam változót a külvilág elõl, így a programot olvasva világos, hogy ezt a kezdeti beállítás után kívülrõl nem szabad módosítani. (A legyártott, belsõ függvény számára persze hozzáférhetõ marad.)

Egyedi függvény-objektumok gyártása

Olyankor lehet szükség arra, hogy egy függvénybõl ne csak egyetlen példány létezzen, amikor valamilyen statikus attribútummal ruházzuk fel a függvényt, amely a hívások között megõrzi az értékét, viszont a program különbözõ helyein más és más attribútum értékre van szükség. Legyen például egy függvényünk, amelyik valamilyen paramétereket kiértékel és ezeknek a függvényében jelzéseket ad ki; mondjuk figyelmeztet, hogy le kell olvasztani a hûtõt. A figyelmeztetés nyelvét be lehet állítani angolra vagy németre. Ha több helyen is szükség van erre a funkcióra és nem mindig mindenhol ugyanazt a nyelvet akarjuk használni, akkor több függvény-objektum példányra van szükségünk. 

def warn_factory():
    def f_warning():
        if f_warning.lang == 'en':
            print('I warn you!')
        elif f_warning.lang == 'de':
            print('Donnerwetter!')

    return f_warning

f_warn_1 = warn_factory()
f_warn_1.lang = 'en'
f_warn_2 = warn_factory()
f_warn_2.lang = 'de'

f_warn_1()
f_warn_2()


# I warn you!
# Donnerwetter!
A gyár függvény persze elvégezheti az attribútum kezdeti beállítását: 
def warn_factory(language):
    def f_warning():
        if f_warning.lang == 'en':
            print('I warn you!')
        elif f_warning.lang == 'de':
            print('Donnerwetter!')

    f_warning.lang = language

    return f_warning

f_warn_1 = warn_factory('en')
f_warn_1.lang = 'en'
f_warn_2 = warn_factory('de')
f_warn_2.lang = 'de'

f_warn_1()
f_warn_2()
