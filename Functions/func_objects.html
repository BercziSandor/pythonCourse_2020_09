A függvények igazi objektumok

Pythonban a függvény "teljes jogú" objektum (first class citizen), azaz úgy viselkedik, mint bármilyen más objektum, pl. egy egész szám vagy egy sztring. Egy függvény-objektum

  • hozzárendelhetõ változókhoz,
  • lehet egy függvény paramétere,
  • visszatérõ értéke,
  • elhelyezhetõ adatstruktúrákban, mint pl. egy list vagy dict,
  • lehetnek attribútumai és metódusai.
A függvény-objektum konstruktora a függvény definíciója, ez akkor fut le, amikor a futtató rendszer a def utasításhoz ér. A függvény definíciója olyan különleges konstruktor, ami azonnal le is fut, amikor a futtató rendszer erre a pontra érkezik (tehát nem kell külön felhívni) és mindjárt hozzá is rendel egy változónevet az így keletkezõ objektumhoz. 
def func():
    pass

print(func, id(func))


# <function func at 0x007A8FA8> 8032168
Keletkezett egy objektum a 0x007A8FA8 memóriacímen, amelynek van azonosítója is, mint minden rendes, becsületes objektumnak (8032168) és egy func nevû változó mutat rá. Az azonosító egyébként a memóriacím decimális formában, amirõl könnyen meggyõzõdhetünk: 
print(func, hex(id(func)))


# <function func at 0X007A8FA8> 0x007a8fa8

Változók hozzárendelése függvényhez

Ha több változóhoz hozzárendeljük a függvényobjektumot, ezek mind ugyanarra az egyetlen objektumra fognak mutatni: 

def func():
    print('Hi, I am func')

f = func # NEM történik függvényhívás!
print(func, id(func))
print(f, id(f))
func()
f()


# <function func at 0x01E98FA8> 32083880
# <function func at 0x01E98FA8> 32083880
# Hi, I am func
# Hi, I am func
Itt az f és a func változók ugyanarra a függvény-objektumra mutatnak; f() és func() hatására ugyanaz a kód fut le. Ami meg szokta lepni az embereket: a függvényre mutató változók egyenrangúak, bármelyiket, akár a definícióban szereplõt is meg lehet semmisíteni, az objektum maga és a többi rá mutató változók továbbra is élnek. 
def func():
    print('Hi, I am func')

f = func
del func
f()


# Hi, I am func
Más modulok függvényeire is ráállíthatunk persze változókat, sõt, az eredetit meg is semmisíthetjük (ami nyilván nagyon rossz ötlet): 
import math

my_sq = math.sqrt
print(my_sq(16))

del math.sqrt   # ⛔ ☠ ⚠ ☢ ☣

print(my_sq(100))


# 4.0
# 10.0

Függvény, mint hívási paraméter (callback)

Egy függvény-objektumot átadhatunk egy másik függvénynek paraméterként: 

def func(funcParam):
    print('I am func, I call the function I received as parameter')
    funcParam()

def func_2():
    print('*** I am func_2')

func(func_2)


# I am func, I call the function I received as parameter
# *** I am func_2
Ezt a mechanizmust használjuk, amikor eseménykezelõket rendelünk hozzá a felhasználói felület elemeihez. Szimuláljuk nagyon leegyszerûsítve: 
def ui(click_func,double_click_func):
    while True:
        answer = input('what do you want? ')
        answer = answer.strip().lower()
        if answer == '':
            break

        if answer == 'c':
            click_func()
        if answer == 'd':
            double_click_func()

def cl():
    print('CLICK!')

def d_cl():
    print('DOUBLE CLICK!')

ui(cl,d_cl)

def cl_2():
    print('click 2')

def d_cl_2():
    print('double click 2')

ui(cl_2,d_cl_2)
Ha a felhasználó c-t gépel be, akkor a click_func hívódik fel, jelen esetben cl; ha d-t, akkor double_click_func, ha bármi mást, akkor nem történik semmi.

♣

Egy másik példát szolgáltat a standard könyvtár sorted függvénye, aminek megadhatunk key kulcsszóval egy függvényt, amelyet az összehasonlításban fog használni. A példában sztringeket rendezünk a hosszuk szerint: 

lst = ['Z','FFF','AA']

lst_2 = sorted(lst)         # "normal" sorting
print(lst_2)
lst_3 = sorted(lst,key=len) # sort by string length
print(lst_3)


# [ 'AA', 'FFF', 'Z']
# [ 'Z', 'AA', 'FFF']
Gyakran elõforduló eset, hogy egy tuple vagy list elemekbõl álló listát kell valahányadik elem szerint rendezni: 
lst = [('Z','A'),('A','K'),('B','Y')]

lst_2 = sorted(lst)   # "normal" sorting
print(lst_2)

lst_3 = sorted(lst,key=lambda elem: elem[1]) # sort by second item
print(lst_3)

def key_func(elem):
    return elem[1]

lst_3 = sorted(lst,key=key_func)
print(lst_3)


# [('A', 'K'), ('B', 'Y'), ('Z', 'A')]
# [('Z', 'A'), ('A', 'K'), ('B', 'Y')]
# [('Z', 'A'), ('A', 'K'), ('B', 'Y')]
A harmadik változat ekvivalens a másodikkal – azok kedvéért tettem bele, akik még idegenkednek a lambda függvényektõl.

Függvény mint visszatérõ érték (gyár-függvények)

Egy függvény-objektum lehet egy másik függvény visszatérõ értéke. Az alábbi kis példában func_factory minden alkalommal visszaad egy belsõ függvényére mutató változót: 

def func_factory():
    def func():
        print('I am func')

    return func

f1 = func_factory()
f2 = func_factory()

print(f1,id(f1))
print(f2,id(f2))

f1()
f2()


# <function func_factory.<locals>.func at 0x02128F60> 34770784
# <function func_factory.<locals>.func at 0x02128F18> 34770712
I am func
I am func
Látható, hogy f1 és f2 nem ugyanaz az objektum, hanem egy függvénynek két külön példánya. Ez például akkor lehet hasznos, amikor attribútummal ruházzuk fel a függvényt és ennek példányonként más értéket akarunk adni (ld. lentebb).

Természetesen nem csak belsõ függvény lehet visszatérõ érték, de ez kiemelten fontos eset, mert ez a closure és ezáltal a dekorátorok alapja.

Az, hogy egy függvény lehet függvényparaméter vagy visszatérõ érték, nem bonyolult fogalom.
Viszont szokatlan.
És a szokatlan dolgok legalább annyi galibát okoznak, mint a bonyolultak.

Függvények adatstruktúrákban

A következõ példában elhelyezünk (különösebb cél nélkül) két függvényt egy szótárban, aztán meghívjuk õket kétféle módon: közvetlenül, illetve egy segédváltozót ráállítva a függvényre. 

import math

d = { 'square_root': math.sqrt, 'log_10': math.log10 }

x = d['square_root'](100)
print(x)
f = d['square_root']
x = f(100)
print(x)

y = d['log_10'](100)
print(y)
f = d['log_10']
y = f(100)
print(y)


# 10.0
# 10.0
# 2.0
# 2.0
Elhelyezhetjük a függvény-objektumokat egy listában vagy egy halmazban is: 
import math

lst = [ math.sqrt, math.log10 ]
# lst = { math.sqrt, math.log10 }
for f in lst:
    print(f(100))


# 10.0
# 2.0
A függvény-listát nyilván használhatjuk egy list comprehension-ben is: 
import math

lst = [ math.sqrt, math.log10 ]

x = [ f(100) for f in lst ]
print(x)


# [10.0, 2.0]
♣
A fentebbi, felhasználói felületet szimuláló kis programban használhatunk például szótárat a callback függvények eltárolására. 
def ui(click_func,double_click_func):
    def no_op(): pass

    func_dict = { 'c': click_func, 'd': double_click_func }
    while True:
        answer = input('what do you want? ')
        answer = answer.strip().lower()
        if answer == '':
            break

        f = func_dict.get(answer,no_op)
        f()

def cl():
    print('CLICK!')

def d_cl():
    print('DOUBLE CLICK!')

ui(cl,d_cl)
Itt definiáltunk egy kis belsõ segédfüggvényt, ami semmit nem csinál, ezt hajtjuk végre minden olyan esetben, amikor nem c vagy d a beadott kód. Kitérõként érdemes megjegyezni, hogy szebb megoldás, ha ehelyett egy lambda függvényt alkalmazunk: 
def ui(click_func,double_click_func):
    def no_op(): pass
    func_dict = { 'c': click_func, 'd': double_click_func }
    while True:
        answer = input('what do you want? ')
        answer = answer.strip().lower()
        if answer == '':
            break

        f = func_dict.get(answer,lambda: None)
        f()

def cl():
    print('CLICK!')

def d_cl():
    print('DOUBLE CLICK!')

ui(cl,d_cl)

Függvények attribútumai

A függvény-objektumnak ugyanúgy adhatunk attribútumokat, mint bármely más objektumnak, akár a függvényen belül, akár rajta kívül. 

def func():
    print('In func; func.x =',func.x)
    func.y = 'BBB'

func.x = 'AAA'
func()
print('In main; func.y =',func.y)


# In func; func.x = AAA
# In main; func.y = BBB
Az attribútum megmarad a hívások között, szolgálhat például arra, hogy számoljuk a függvényhívások számát. 
def func():
    func.num_calls += 1
    print(f'in func number of calls: {func.num_calls}')

func.num_calls = 0
func()
func()
print('total:',func.num_calls)


# in func number of calls: 1
# in func number of calls: 2
# total: 2
Elegánsabb persze ebben az esetben, ha a függvényen belül inicializáljuk az attribútumot: 
def func():
    if not hasattr(func,'num_calls'):
        func.num_calls = 0

    func.num_calls += 1
    print(f'in func number of calls: {func.num_calls}')

func.num_calls = 0
func()
func()
print('total:',func.num_calls)


# in func number of calls: 1
# in func number of calls: 2
# total: 2
Fontos megjegyezni, hogy akárhány változó mutat a func függvényre, az objektum egy és ugyanaz – és ezen belül az attribútum értéke is. Ha több objektum-példányra van szükségünk, amelyekben különbözhetnek ugyanazon attribútum értékei, akkor a fentebb bemutatott gyár-függvényt tudjuk használni: 
def func_factory():
    def func():
        if not hasattr(func,'num_calls'):
            func.num_calls = 0

        func.num_calls += 1
        print(f'in func number of calls: {func.num_calls}')

    return func

f1 = func_factory()
f2 = func_factory()
f1()
f1()
f2()
print('total:',f1.num_calls + f2.num_calls)


# in func number of calls: 1
# in func number of calls: 2
# in func number of calls: 1
# total: 3
Az attribútumok helyett általában closure használható, amelyrõl külön fejezet szól.

Függvények metódusai

A függvény-objektumot felruházhatjuk metódusokkal is. 

def func():
    print('I am func, I call my method')
    func.method()

def func_2():
    print('*** I am func_2')

def func_3():
    print('--- I am func_3!')

func.method = func_2
func()
func.method = func_3
func()


# I am func, I call my method
# *** I am func_2
# I am func, I call my method
# --- I am func_3
Így alkalmazni tudjuk a Strategy tervezési mintát; a változtatható metódus lehet pl. egy útvonaltervezõ algoritmus. Más-más algoritmust akarunk használni autós, gyalogos, kerékpáros, tömegközlekedési útvonaltervezéshez; mindegyik algoritmust más-más függvényben valósítunk meg, ami által mindegyik külön fejleszthetõ lesz és nem keverednek a különbözõ megoldások áttekinthetetlen kuszasággá.
