Definíció

A yield from utasítással delegálni tudjuk az iterálást egy generátornak. Amikor csak egyirányú adatforgalom van (nincs send(), csak a felhívott generátor ad vissza értékeket), akkor ez minimális egyszerûsítést jelent. Egy nagyon egyszerû példa: 

def sub_gen():
    print('sub_gen started')
    yield('1. sub_gen yield')
    yield('2. sub_gen yield')
    yield('3. sub_gen yield')

def main_gen():
    print('main_gen started')
    sub_it = sub_gen()
    yield from sub_it

main_it = main_gen()
for i in main_it:
    print(i)


# main_gen STARTED
# sub_gen STARTED
# 1. sub_gen yield
# 2. sub_gen yield
# 3. sub_gen yield
main_gen csak továbbadta a sub_gen-tõl kapott értékeket a hívónak. Ha nem létezne a yield from utasítás, akkor mindössze ennyivel bonyolódna a program: 
def main_gen():
    print('main_gen STARTED')
    sub_it = sub_gen()
    for i in sub_it:
        yield (i)
A legegyszerûbb esetben tehát nem történik más, mint hogy a yield from utasítás "végigjárja" a belsõ generátort next() utasításokkal és a kapott értékeket továbbadja a hívónak. Megspórolunk egy for ciklust, ami egyszerûsíti a program szerkezetét, de nem sokkal.

Egy másik szemléltetõ példácska: 

def f1():
    yield from 'ABC'

for i in f1():
    print(i)


# A
# B
# C

yield from kétirányú adatforgalomnál

A yield from elõnye akkor válik világossá, amikor nemcsak fogadni akarjuk a generátor által küldött adatot, hanem send() segítségével küldeni is szeretnénk neki. A következõ példában a "belsõ" generátor mindig hozzáad a kapott értékekhez 1000-et és így adja vissza a hívónak; a delegáló generátor létrehozza a belsõt és csak közvetíti az adatokat a hívó és a belsõ generátor között.

Elõször írjuk meg a belsõ generátort és használjuk közvetlenül. 

def sub_gen():
    print('sub_gen STARTED')
    received_in_send = yield(None)  # respond to first next()
    while True:
        if received_in_send is None:
            return_value = None
        else:
            return_value = received_in_send + 1000

        received_in_send = yield(return_value)
A yield(None)-ra azért van szükség, mert indításkor egy generátornak csak None-t lehet küldeni – erre õ célszerûen None-t küld vissza; amikor ez megtörtént, akkor már tud fogadni send()-el küldött számokat. Minden generátort mindig be kell kurblizni egy elsõ next() hívással. 
sub_it = sub_gen()
x = next(sub_it)  # start the generator
print('caller received after first next:',x)
for i in range(50,53):
    x = sub_it.send(i)
    print('caller sent:',i,'received:',x)


# sub_gen STARTED
# caller received after first next: None
# caller sent: 50 received: 1050
# caller sent: 51 received: 1051
# caller sent: 52 received: 1052
Készítsünk most egy "külsõ" generátort, ami csak közvetíti az adatforgalmat. 
def main_gen():
    print('main_gen STARTED')
    sub_it = sub_gen()
    yield from sub_it

main_it = main_gen()
x = next(main_it)  # start the generator
print('caller received after first next:',x)
for i in range(50,53):
    x = main_it.send(i)
    print('caller sent:',i,'received:',x)


# main_gen STARTED
# sub_gen STARTED
# caller received after first next: None
# caller sent: 50 received: 1050
# caller sent: 51 received: 1051
# caller sent: 52 received: 1052
Látható, hogy main_gen-nek semmit nem kell csinálnia ahhoz, hogy mindkét irányban átmenjen rajta az adat. A yield from utasítás mágikusan gondoskodik errõl. 

    +--------------+             +------------+             +---------+
    |              | send()  ==> |  main_gen  | send () ==> |         |
    | Hívó program | <== yield() |            | <== yield() | sub_gen |
    |              |             | yield from |             |         |
    +--------------+             +------------+             +---------+
Éppen ez az egyik oka, amiért ez nehéz fogalom: egy egyszerû jelölés bonyolult, a háttérben lezajló kétirányú adatforgalmat ír le. Ráadásul a név sem igazán szerencsés, azt sugallja, hogy csak yield utasítások hajtódnak végre; a delegate megnevezés találóbb lenne.

Valósítsuk meg az átjátszóállomásként szolgáló generátort úgy, hogy nem használunk yield from utasítást. 

def main_gen():
    print('main_gen STARTED')
    sub_it = sub_gen()
    received_from_sub = next(sub_it)  # start the sub generator
    received_from_caller = yield(received_from_sub)

    while True:
        received_from_sub = sub_it.send(received_from_caller)
        received_from_caller = yield(received_from_sub)
Látható, hogy ez már lényegesen bonyolultabb – és akkor arról még nem is szóltunk, hogy a yield from a kivételeket is helyesen kezeli, azok számára is teljesen transzparens.

Gyakori eset, hogy nem a külsõ generátor belsejében hozzuk létre a belsõt, hanem paraméterként adjuk át neki: 

def main_gen(subIt):
    print('main_gen STARTED')
    yield from subIt
 
sub_it = sub_gen()
main_it = main_gen(sub_it)

x = next(main_it)  # start the generator
print('caller received after first next:',x)
for i in range(50,53):
    x = main_it.send(i)
    print('caller sent:',i,'received:',x)
