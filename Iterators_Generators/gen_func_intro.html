Definíció

GENERÁTOR: iterátor, amelyet nem osztállyal, hanem olyan függvénnyel valósítunk meg, amelyben legalább egy yield utasítás van (return helyett). 

def my_first_generator():
    yield(100)
    yield(110)
    yield(120)

gen = my_first_generator()
print(gen)


# <generator object my_first_generator at 0x022393F0>
Tehát a generátor függvény felhívásakor csak elkészült az iterátor. 
print(next(gen))
print(next(gen))
print(next(gen))
print(next(gen))


# 100
# 110
# 120
# Traceback (most recent call last):
#   File "test.py", line 25, in 
#     print(next(gen))
# StopIteration

Ha yield(100) helyett return 100 lenne, akkor mindig 100 lenne a kimenet. A generátor-függvény megjegyzi a belsõ állapotát és a következõ next()-nél onnan folytatja a végrehajtást, ahol a yield-del abbahagyta.

Amikor a generátor "kimerült", azaz nem tud több értéket szolgáltatni, akkor a next() hatására StopIteration kivételt dob. 


  HÍVÓ           GENERÁTOR

next()  ==>    (nincs több elem)

        <==    StopIteration
A hívó így értesül arról, hogy kimerült a generátor.

for ciklusban ugyanúgy lehet használni, mint bármely iterátort: 

gen = my_first_generator()
for x in my_first_generator():
    print(x)


# 100
# 110
# 120


for i,x in enumerate(my_first_generator()):
    print(i,x)


# 0 100
# 1 110
# 2 120
A for ciklus kiadja a next() hívásokat és amikor StopIteration kivételt kap, akkor befejezi a ciklust.

Mivel ez a generátor számokat produkál, ezeket összegezni is tudjuk: 

print(sum(my_first_generator()))


# 330
A sum() függvény mindaddig hívja a next() függvényt, míg ki nem merül a generátor és a visszakapott elemeket összegzi. A join() függvény segítségével össze tudjuk fûzni a generátor által szolgáltatott elemeket: 
def my_gen():
    yield('AAA')
    yield('bbb')
    yield('CCC')

print('|'.join(my_gen()))


# AAA|bbb|CCC

Összehasonlítás objektum-iterátorral

Nézzük meg, egy osztállyal hogyan tudnánk létrehozni ugyanezt az iterátort. 

class my_first_gen_class():
    def __init__(self):
        self.value = 100

    def __iter__(self):
        return self

    def __next__(self):
        if self.value <= 120:
            ret = self.value
            self.value += 10
            return ret
        else:
            raise StopIteration

gen = my_first_gen_class()
print(gen)

for x in gen:
    print(x)


# <__main__.my_first_gen_class object at 0x02307490>
# 100
# 110
# 120

Sok segéd-kódot kellett létrehoznunk, nehezebben áttekinthetõ az egész.

Generátor elõnye osztály-iterátorokkal szemben:

● Sok esetben egyszerûbb, mint az erre a célra készített osztállyal felépített iterátor.
● Ha legalább egy yield-et elhelyezünk egy függvényben, ezzel automatikusan generátorrá válik.

Persze ha más funkcionalitásra (metóduskra) és megjegyzett állapotváltozókra (tagváltozókra, attribútumokra) is szükség van, akkor már az objektumos megoldás lesz egyszerûbb.

A generátor memóriája

A generátor függvény alkalmas arra, hogy az inicializáláskor kapott argumentumokat és a késõbb keletkezõ lokális változók értékét megjegyezze. Ebben hasonlít egy objektumra, ami szintén meg tudja õrizni a konstruktorban kapott paramétereket és a késõbb keletkezõ tagváltozókat is. 

def my_second_generator(param_1, param_2):
    local_1 = 100
    for _ in range(3):
        print('in generator:',param_1,param_2,local_1)
        local_1 += 10
        yield(1000 + local_1)

gen = my_second_generator('AAA', 'BBB')
for i in gen:
    print('*** received from generator:',i)


# in generator: AAA BBB 100
# *** received from generator: 1110
# in generator: AAA BBB 110
# *** received from generator: 1120
# in generator: AAA BBB 120
# *** received from generator: 1130
Egy érdekes különbség: az objektum csak akkor "emlékszik" az __init__-ben kapott paraméterekre, ha azokat eltárolja tagváltozókban, míg a generátornak nem kell változókba bemásolnia az indításkor kapott paramétereket ('AAA','BBB').

Return a generátor függvényben

Eddig olyan eseteket láttunk, amikor a függvény törzsének végéhez értünk végrehajtás közben - ekkor a Python futtató rendszer automatikusan StopIteration kivételt dob Ha for ciklusban mûködtetjük az iterátort, akkor az lenyeli ezt a kivételt és véget ér a ciklus.

Ugyanez történik, ha return-nel lépünk ki a generátor-függvénybõl. 

def my_third_generator():
    yield 100
    yield 200
    return 'Goodbye!'

    print('Never reach this')

gen = my_third_generator()
for i in gen:
    print('*** received from generator:',i)


*** received from generator: 100
*** received from generator: 200
A return által visszaadott érték a StopIteration paramétereként megy át a hívóhoz: 
gen = my_third_generator()
while(True):
    try:
        x = next(gen)
        print('*** received from generator:',x)
    except StopIteration as e:
        print('Caught StopIteration exception:',e)
        break


# *** received from generator: 100
# *** received from generator: 200
# Caught StopIteration exception: Goodbye!

StopIteration-t dobunk a generátor függvényben

Nem jó ötlet. 

def my_third_generator():
    yield 100
    yield 200
    raise StopIteration

    print('Never reach this')


gen = my_third_generator()
for i in gen:
    print(i)
Ha 3.7-nél régebbi Python verziónk van, akkor ez lesz a kimenet: 

# 100
# 200
Ha viszont legalább 3.7-es, akkor ez: 

# 100
# 200
#
# Traceback (most recent call last):
#   File "tst.py", line 4, in my_third_generator
#     raise StopIteration
# StopIteration
#
# The above exception was the direct cause of the following exception:
#
# Traceback (most recent call last):
#   File "tst.py", line 10, in 
#     for i in gen:
# RuntimeError: generator raised StopIteration
A PEP 479 szerint mûködnek már ezek a verziók, azaz minden generátorfüggvényben keletkezõ StopIteration-t futási hibává konvertál a Python. Szerintem ez logikus, mert kevesebb így a lehetõség félreértésekre. A StopIteration-t a futtató rendszer használja, nem jó, ha más célokra is alkalmazni akarjuk.

TANULSÁG

Ha be akarjuk fejezni a generátor függvény futását, argumentum nélküli return-t használjunk.

REJTVÉNYEK

①

Egy olyan generátorra van szükségünk, amely egy számsorozatot kap bemenetként és a kimeneten kiadja a számokat úgy, hogy a párosakhoz 100-at ad hozzá, a páratlanakhoz pedig 200-at. Egy teszteset és a várt kimenet: 

for x in my_gen(range(1,5)):
    print(x)


# 201
# 102
# 203
# 104
A generátorunk és a teszteset: 
def my_gen(inputNumbers):
    for n in inputNumbers:
        if n % 2 == 0:
            yield(n + 100)

        yield(n + 200)

for x in my_gen(range(1,5)):
    print(x)


# 201
# 102
# 202
# 203
# 104
# 204
Megjelentek felesleges számok a kimeneten.

Vajon miért?

A yield paraméterét visszaadja a generátor a hívónak és várja a következõ next()-et. Amikor ez megérkezik, a yield utáni utasítással folytatja a végrehajtást. Amikor páros szám érkezik, akkor hozzáad 100-at és várakozik; a következõ next() hívásnál itt folytatja, azaz yield(n + 200)-nál.
Hogyan lehet kijavítani a generátort?
Helyesen: 
def my_gen(inputNumbers):                     def my_gen(inputNumbers):
    for n in inputNumbers:                        for n in inputNumbers:
        if n % 2 == 0:                                if n % 2 == 0:
            yield(n + 100)       VAGY:                    yield(n + 100)
        else:                                             continue
            yield(n + 200)
                                                      yield(n + 200)

②

Adott egy generátor: 

def my_gen():
    yield(200)
    yield(300)
    yield(500)
Elõ akarunk állítani egy listát, amelynek az elemei a my_gen által szolgáltatott számok, osztva az összegükkel. A várt kimenet: 

# [0.2, 0.3, 0.5]
Az elsõ megoldás és a kimenet: 
g = my_gen()
x = [ i / sum(g) for i in g ]
print(x)


# [0.25]
Arra gyorsan rájövünk, hogy mi a probléma: a sum() függvény kimeríti a generátort, ezért kapunk csak egyetlen elemet.

De miért ez a kimenet?

Elindul a for ciklus, veszi az elsõ elemet a generátorból, ez 200. Ezután meghívjuk a sum() függvényt, ami összeadja a maradék elemeket, az eredmény 800.

Mi a jó megoldás?

Elõször kiszámítjuk az összeget, utána újra létrehozzuk a generátort: 
g = my_gen()
s = sum(g)
g = my_gen()
x = [ i / s for i in g ]
print(x)


# [0.2, 0.3, 0.5]
