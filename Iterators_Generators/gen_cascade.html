Mit jelent a kaszkádba kapcsolás?

A generátoroknak nagy elõnye, hogy könnyen összekapcsolhatók egymással. Ha egy generátort paraméterként átadunk egy másiknak, akkor a hívó egyenként ki tudja olvasni az értékeit és feldolgozás után yield-ek segítségével tovább tudja adni egy következõnek. Amely aztán ugyanígy járhat el...

Ilyen módon egymással nagyon könnyen kombinálható és újrafelhasználható elemek jönnek létre, a program áttekinthetõ lesz. A "hagyományos" megoldásnál az egyes függvények eredményeit közbülsõ tárolókban kell elhelyeznünk, a következõ függvény innen olvassa ki azokat.

Egy értelmetlen, de cserébe egyszerû kis példa

def gen_1(numbers):  # numbers: an iterator producing numbers
    for n in numbers:
        yield (2 * n)

def gen_2(numbers):  # numbers: an iterator producing numbers
    for i in numbers:
        yield (i + 100)

nums_source = range(1,5) # source iterator producing numbers
g1 = gen_1(nums_source)
g2 = gen_2(g1)

for n in g2:
    print(n, end=' ')
print()


# 102 104 106 108
Figyeljük meg, hogy a végrehajtás a lánc végérõl kezdõdik! A g2 iterátor kap egy next() hívást a for ciklustól, erre õ kiad egy next() hívást a paraméterként kapott g1 iterátornak, mire az kiad egy next() hívást a paraméterként kapott range iterátornak.

Sehol nem állítunk elõ közben listát, minden iterátor csak egyenként foglalkozik az elemekkel.

Az egész nagyon hasonlít az eredetileg a UNIX-ban bevezetett pipeline elrendezéshez: amikor programok a standard bemenetrõl olvasnak és az eredményt a standard kimenetre írják, akkor könnyen kaszkádba köthetõk.

Egy valóságosabb példa

Egy szövegfájlban mérési adatok vannak, a sorok pontosvesszõvel elválasztott mezõket tartalmaznak. Vannak üres (csak fehér karaktereket tartalmazó) sorok és megjegyzés-sorok, amelyekben az elsõ látható karakter egy # – mindkét fajta sort ki kell szûrni. Az elsõ mezõ egy azonosító, a második egy numerikus érték. Vannak hibás sorok:

  • csak egy mezõ van
  • a második mezõben nem szám van
és az azonosítók is lehetnek kisbetûvel vagy nagybetûvel írva - azt akarjuk, hogy írásmódtól függetlenül mindig ugyanazt azonosítson minden sztring. Vannak olyan sorok is, amelyekben kettõnél több mezõ van, itt a többlet-mezõket egyszerûen figyelmen kívül kell hagyni.

Elõfordulhatnak olyan sorok, ahol a szám nem tizedespontot, hanem tizedesvesszõt tartalmaz.

Az üres és hibás sorokat ki kell hagyni, a többiben egyforma formátumúra kell hozni az azonosítókat és képezni kell egy azonosító - szám párokból álló listát. A feladat tehát tulajdonképpen adattisztítás.

A bemeneti fájl pl. ilyen: 

# comment

   # and again

AAA; 15
  bbb ;  25.2;
Ccc; 33;third column;
ddd; 99 and some text
eee

fff ; 32,3
ggg;
aaa; 45
ccc; 22;
Aaa; 9;
Elõször elõállítunk egy olyan generátort, amely eltávolítja az üres és a megjegyzés-sorokat. 
def nonempty_lines(inputLines):
    '''Generator that removes lines that contain only whitespace and comment-lines, too'''
    for line in inputLines:
        line = line.strip()
        if len(line) == 0:
            continue
        if line.startswith('#'):  # comment line
            continue

        yield(line)
Elsõ teszt: egy egyszerû listával próbáljuk ki. 
test_lines = [' aaa','   ','  # bbb','ccc ']
g_lines = nonempty_lines(test_lines)
for line in g_lines:
    print(line)


# aaa
# ccc
A g_lines névben a g arra emlékeztet, hogy ez egy generátor.

Most olvassuk be a fenti tesztfájlt, legyen az a forrás. 

with open('infile.txt','r') as f:
    g_lines = nonempty_lines(f)
    for line in g_lines:

Most megírjuk az a generátort, amely számmá alakítja a második mezõt (ha van), nagybetûssé az elsõt és ahol ez sikerült, ott kiadja az elsõ két mezõt. 
def lines_with_number(inputLines):
    '''Converts 2nd field to number, discards false lines, turns names to uppercase'''
    for line in inputLines:
        lst = line.split(';')
        if len(lst) < 2:   # only one field
            continue

        lst = lst[:2]
        lst[0] = lst[0].strip().upper()
        num_fld = lst[1]
        try:
            num = float(num_fld)
        except ValueError:
            pass
        else:
            lst[1] = num                    # could convert to number
            yield (lst)
            continue                        # important! without this the program
                                            # will resume at the next line next time

        num_fld = num_fld.replace(',','.')  # maybe comma instead of dot?
        try:
            num = float(num_fld)
        except ValueError:
            continue
        else:
            lst[1] = num                    # yes, it was
            yield lst
Az elõbbi tesztprogramot csak egyetlen sorral kell kiegészítenünk. 
with open('infile.txt','r') as f:
    g_lines = nonempty_lines(f)
    g_filtered = lines_with_number(g_lines)
    for line in g_filtered:
        print(line)

# ['BBB', 25.2]

Vegyük észre azt is, hogy ha csak annyit teszünk, hogy a for ciklusba g_filtered helyett g_lines-t írunk, akkor ismét az elsõ lépést tudjuk tesztelni. g_filtered ilyenkor csak készen áll, de nem mûködik.

Most már van egy megtisztított és egységes formára hozott adatsorozatunk, amivel sokmindent tudunk kezdeni. Tegyük fel, hogy még mindig nem akarjuk az adatokat egy listában elhelyezni, ragaszkodunk a generátorokhoz.

— Vehetjük a második oszlop összegét. 

with open('infile.txt','r') as f:
    g_lines = nonempty_lines(f)
    g_filtered = lines_with_number(g_lines)
    second_column = (row[1] for row in g_filtered)
    print(sum(second_column))


# 181.5
A second_column is egy generátor (generátorkifejezés)!

Figyelem: még mindig nem kellett semelyik lépésben sem a teljes adatsorozatot tárolni! A sum() függvény is egyenként kapja meg az iterátortól az értékeket. 


sum() --> next() --> second_column --> next() --> g_filtered --> next() --> g_lines --> next() --> f
Ha a sum() függvény helyére max()-ot írunk, akkor megkapjuk a második oszlop maximumát; vagy képezhetjük például az oszlop átlagát: 
from statistics import mean
...
    print(mean(second_column))
— Most iktassunk be egy szûrõt, ami csak az olyan sorokat engedi át, amelyek nagyobbak egy bizonyos értéknél: 
def filter_greater(matrix, threshold):
    for row in matrix:
        if row[1] < threshold:
            continue
        yield(row)

with open('infile.txt','r') as f:

— Készíthetünk egy dict-et, amely az egyforma azonosítójú elemekhez tartozó számok összegét tartalmazza: 
def sum_dict(matrix):
    d = {}
    for row in matrix:
        if row[0] not in d:
            d[row[0]] = 0
        d[row[0]] += row[1]

    return d

with open('infile.txt','r') as f:

    g_filtered = lines_with_number(g_lines)

A generátor-lánc elejét bõvítjük

Eddig egyetlen fájllal dolgoztunk, annak a feldolgozásába kapcsoltunk be újabb és újabb szûrõ-generátorokat. Nézzük meg, mit tehetünk, ha több fájlunk van, amelyeknek a tartalmából egyetlen listát szeretnénk készíteni. Legyen pl. a másik fájl neve other_file.txt és a tartalma: 

XXX_OTHER_FILE;2000
YYY_OTHER_FILE;3000
Ezt a kimeneti listát szeretnénk elõállítani a két fájlból: 

# ['AAA', 15.0]
# ['BBB', 25.2]
# ['CCC', 33.0]
# ['FFF', 32.3]
# ['AAA', 45.0]
# ['CCC', 22.0]
# ['AAA', 9.0]
# ['XXX_OTHER_FILE', 2000.0]
# ['YYY_OTHER_FILE', 3000.0]
Elõször is tegyük bele az egyetlen fájlhoz tartozó szûrõ-funkciókat egy generátor függvénybe. 
def format_file(fileObj):
    with fileObj:
        g_lines = nonempty_lines(fileObj)
        g_filtered = lines_with_number(g_lines)
        for line in g_filtered:
            yield(line)
A fájl elérési útvonalakat most egy listából fogjuk kiolvasni, ebbõl elõállítunk fájl-objektumok sorozatát (megnyitjuk a fájlokat), majd mindegyiken végrehajtjuk a szûrést. 
filepaths = ['infile.txt','other_file.txt']
g_f_objs = (open(f,'r') for f in filepaths)
g_formatters = (format_file(f_obj) for f_obj in g_f_objs)
for fmt in g_formatters:
    for line in fmt:
        print(line)
A kettõs ciklust persze elhelyezhetjük ízlés szerint egy generátor-kifejezésben is: 
filepaths = ['infile.txt','other_file.txt']
g_f_objs = (open(f,'r') for f in filepaths)
g_formatters = (format_file(f_obj) for f_obj in g_f_objs)
g_lines = (line for fmt in g_formatters for line in fmt)
for line in g_lines:
    print(line)
Ha valakinek olvashatóbb így, akkor persze írhatja több sorba is: 
g_lines = (line
           for fmt in g_formatters
               for line in fmt)

Amikor return kell yield helyett

REJTVÉNY

Tegyük fel, hogy a fájl megnyitását bele akarjuk tenni egy függvénybe (hamarosan látni fogjuk, hogy ez miért lehet célszerû). Logikusnak látszik, hogy a megnyitott fájlobjektumot yield segítségével adjuk vissza, hiszen annyira szeretjük a generátorfüggvényeket. Próbáljuk is ki mindjárt az új függvényünket, olvassunk egy sort a fájlból: 

def open_file(filePath):
    f_obj = open(filePath,'r')
    yield(f_obj)

f = open_file('infile.txt')
f.readline()


# Traceback (most recent call last):
#   File "small.py", line 87, in 
#     f.readline()
# AttributeError: 'generator' object has no attribute 'readline'
Ha a yield helyett return-t használunk: 
def open_file(filePath):
    f_obj = open(filePath,'r')
    return f_obj
akkor jól mûködik.

Vajon miért?

Amikor egy generátor függvényt lefuttatunk, akkor csak egy generátort kapunk, a benne lévõ kód még nem aktiválódik! Ez a sor: 
f_obj = open_file('infile.txt')
csak létrehozza azt a generátort, ami aztán szolgáltat egy fájlobjektumot, amikor majd egy next()-et végrehajtunk rajta. Mivel erre nem számítottunk, a változónév is félrevezetõ, g_f_obj jobb lenne, hogy jelezze: itt nem egy fájlobjektumról, hanem egy generátorról van szó.
A függvényünket így használjuk a generátor-láncban: 
filepaths = ['infile.txt','other_file.txt']
g_f_objs = (open_file(f) for f in filepaths)
g_formatters = (format_file(f_obj) for f_obj in g_f_objs)
g_lines = (line for fmt in g_formatters for line in fmt)
for line in g_lines:
    print(line)

Különféle fájlok megnyitása

Tegyük fel, hogy nemcsak egyszerû szövegfájlokat akarunk feldolgozni, hanem gzip-pel tömörített szövegfájlokat is. Ehhez csak az open_file() függvényt kell megváltoztatnunk; attól függõen használunk más-más módszert, hogy a fájl .gz kiterjesztésû-e. 

import gzip

def open_file(filePath):
    if filePath.lower().endswith('.gz'):
        f_obj = gzip.open(filePath,'rt')
    else:
        f_obj = open(filePath,'r')

    return(f_obj)
A gzip esetében a nyitási paraméterbõl nem szabad lehagyni r mellõl a t-t, különben bináris módban nyílik meg a fájl, azaz bájtsorozatot és nem karaktersorozatot kapunk. A szövegfájlok megnyitásánál elhagyhatjuk ezt a kapcsolót, mert ott a text módus az alapértelmezett.

Tegyük fel, hogy van egy third_file.txt bemeneti fájlunk is, amelyet egy third_file.gz-be tömörítettünk össze; a tartalma:

REJTVÉNYEK

A fenti programban az utolsó sorban a maximumot és a minimumot is ki akarjuk íratni: 

print(max(second_column), min(second_column))
Hibajelzést kapunk: 

# Traceback (most recent call last):
#   File "test.py", line 94, in 
#     print(max(second_column), min(second_column))
# ValueError: min() arg is an empty sequence
Miért?
A max() függvény "kimeríti" a teljes iterátor-szekvenciát, kiolvas belõle minden elemet. Mire a min() függvény sorra kerül, az iterátor üres, a min() függvény pedig nulla darab elemmel nem tud mit kezdeni.

Úgy próbálunk segíteni a problémán, hogy két darab iterátort hozunk létre a második oszlop elõállítására: 

filepaths = ['infile.txt','other_file.txt','third_file.gz']
g_f_objs = (open_file(f) for f in filepaths)
g_formatters = (format_file(f_obj) for f_obj in g_f_objs)
g_lines = (line for fmt in g_formatters for line in fmt)
second_column = (row[1] for row in g_lines)
second_col_2 = (row[1] for row in g_lines)
print(max(second_column), min(second_col_2))
A hibajelzés változatlan marad.

Miért?

Hiába készítünk két iterátort a lánc végére, a láncban a többi iterátor változatlanul ugyanaz – amiket a max() függvény már kiovasott. 
min -- second_col_2  --+
                       |
                       +-- g_lines -- g_formatters -- g_f_objs -- filepaths
                       |
max -- second_column --+
Ha az egész iterátor-láncot megdupláznánk, akkor persze mûködne a dolog, de ez nem volna sem szép, sem gyors megoldás – a fájl beolvasásától kezdve mindent kétszer végig kellene csinálni. És ha még az átlagra is kíváncsiak vagyunk, akkor háromszor...

Kétféle módon tudjuk megoldani a problémát.

① Az egyik módszer, hogy valamelyik ponton listává vagy tuple-lá alakítjuk a sorozatunkat: 

filepaths = ['infile.txt','other_file.txt','third_file.txt.gz']
g_f_objs = (open_file(f) for f in filepaths)
g_formatters = (format_file(f_obj) for f_obj in g_f_objs)
g_lines = (line 
           for fmt in g_formatters
                for line in fmt)
second_column = [row[1] for row in g_lines]
# second_column = tuple(row[1] for row in g_lines)

print(max(second_column), min(second_column))


# 3000.0 9.0
Ha nagy mennyiségû adatunk van, akkor ez az a pont, ahonnan sok memóriára van szükség.

② Megmaradunk a generátoroknál és írunk egy kis függvényt, ami kiszámítja a maximumot és a minimumot. 

def min_max(numbers):
    e = next(numbers)
    min_val = e
    max_val = e
    for n in numbers:
        if n < min_val:
            min_val = n
            continue
        if n > max_val:
            max_val = n
    return (min_val, max_val)

filepaths = ['infile.txt','other_file.txt','third_file.txt.gz']
g_f_objs = (open_file(f) for f in filepaths)
g_formatters = (format_file(f_obj) for f_obj in g_f_objs)
g_lines = (line 
           for fmt in g_formatters
                for line in fmt)
second_column = (row[1] for row in g_lines)

min_val, max_val = min_max(second_column)
print(max_val, min_val)
