Python letöltése

A hivatalos Python oldalról tudjuk a kívánt Python verziót letölteni [1]. A gépünkön lehet több verzió is egyidejűleg, amire például azért lehet szükség, mert egyik projektünkben használhatunk egy régebbi, egy másikban egy újabb verziót.

A 3.X és a 2.X verziók között szándékoltan nagy és inkompatibilis különbségek vannak; a 2-es verzió hivatalos támogatása megszűnt, ennek ellenére persze sok helyen még használatban van.

A 3-as verzión belül a 3.6-os az, amelybe az átlagosnál több (és hasznos) újdonság került be. Nyilván ha tehetjük, akkor a legfrissebb stabil verzióval érdemes dolgozni.

Python programok közvetlen futtatása

A közvetlen szó itt azt jelenti, hogy nem valamilyen integrált fejlesztő környezetben (IDE, mint pl. a VS Code) végezzük a Python programunk futtatását, hanem parancssorból.

python.exe helyének specifikálása

Ha python.exe könyvtárát hozzáadjuk a PATH környezeti változóhoz, akkor a parancssorba elegendő annyit írnunk, hogy python. Ha ezt nem tesszük meg, például azért, mert több verzió is van a gépünkön és pontosan specifikálni akarjuk, melyik fusson, akkor ki kell írnunk a teljes elérési útvolnalat, (ha az útvonalban szóközök is előfordulhatnak, akkor idézőjelek közé kell tennünk): 

c:\Users\NMI\AppData\Local\Programs\Python\Python36-32\python.exe
"c:\Users\NMI\AppData\Local\Program Files\Python\Python36-32\python.exe"
                                   ⇈
A következőkben az egyszerűség kedvéért nem írom ki az elérési útvonalat.

Parancsok interaktív futtatása [2] [4]

Ha a parancssori terminált elindítottuk (Windows alatt: cmd), akkor ebben felhívhatjuk a fentebb megadott módon a python programot; ekkor megjelenik az interaktív mód promptja (rendszerint három > jel) és beírhatjuk a parancsokat, amelyek azonnal végrehajtódnak. A kifejezéseket leírhatjuk print() utasítás nélkül is, amikor ENTER-t ütünk, meg fogjuk kapni a kifejezés értékét.

A Python interpreter leállítását quit() paranccsal, Ctrl-Z (Windows) vagy Ctrl-C (Linux) kombinációval érhetjük el.

Ha beírtunk egy sort és ENTER-t nyomunk, kiértékelődik a sor. Olyan utasításoknál, amelyeknél folytatást vár az interpreter (pl. egy for ciklus feje), az ENTER hatására a következő sorra ugrik és a prompt megváltozik (rendszerint három pontra): Ilyenkor nem szabad megfeledkeznünk a megfelelő indentálásról; az utolsó sor után kitöröljük az indentálást és ENTER-t ütünk, ekkor végrehajtódik a kód. 

python
>>> for i in range(3):
...   print(i)
...   100 + i
...
0
100
1
101
2
102

Ez a megoldás nyilván nehézkes, hiszen mindig újra be kell gépelnünk vagy másolnunk a parancsokat, ezért ritkán használjuk.

Szkript (program) futtatása [3] [4]

Amikor az utasításokat egy .py kiterjesztésű fájlba írjuk abból a célból, hogy ezt majd végrehajtassuk a Python interpreterrel, akkor a fájlt szkriptnek szoktuk nevezni – van, aki szerint a megnevezés nem szerencsés, inkább a program lenne precízebb név. Tegyük fel, hogy egy my_first.py nevű fájlba ezt írtuk: 

i = 10
print('i:', i)
i = 20
Ekkor ezt így tudjuk végrehajtatni: 
python my_first.py
Figyeljük meg, hogy a teljes fájlnevet ki kellett írnunk, kiterjesztéssel együtt. Persze használhajuk a standard bemenet átirányítását is: 
python < my_first.py

Szkript futtatása interaktívan [4]

Nem túl gyakran, de előfordul, hogy a szkript lefutása után még további parancsokat akarunk begépelni; azt szeretnénk szimulálni, mintha a fájl tartalmát közvetlenül írtuk volna be. Ilyenkor a -i kapcsolóval kell felhívnunk az interpretert: 

python -i my_first.py
A képernyőn az interaktív promptot (>>>) fogjuk látni és pl. kiírathatjuk az i változó tartalmát: 
i: 10   # ezt írta ki a szkript
>>> i   # ezt gépeltük be
20
>>>
Az interaktív mód hibakeresésnél lehet hasznos; globális változók tartalmát ki tudjuk íratni a szkript lefutása után.

Modul futtatása [4] [5] [6]

A -m kapcsoló segítségével le tudunk futtatni egy Python modult, pl.: 

python -m pip numpy
python -m cProfile my_first.py
python -m trace -t my_first.py
python -m filecmp dir_1 dir_2
Figyeljük meg, hogy a modul nevéhez nem írunk semmilyen kiterjesztést.

Bemenet és kimenet átirányítása

Ha a szkriptünk a standard bemenetről olvas és azt akarjuk, hogy fájlból jöjjön a bemenet, akkor így járunk el: 

python my_first.py < input.txt
A standard kimenetre – tipikusan print() segítségével – írtakat így tudjuk egy fájlba átirányítani: 
python my_first.py > output.txt
A standard hibakimenetre (stderr) írtakat, a hibajelzéseket pedig így: 
python my_first.py 2> error.txt
Windows alatt a 2 és a > között nem lehet szóköz.

Futtatás Notepad++-ban

A Run menü Run almenüjét választjuk ki (F5), a felugró ablakba ezt írjuk: 

python.exe -i "$(FULL_CURRENT_PATH)"
szükség esetén python.exe teljes elérési útvonalával, majd elmentjük ezt a parancsot pl. pyrun néven. Mentéskor billentyűkombinációt is hozzá tudunk rendelni – vigyázzunk, hogy ez ne ütközzön már meglévővel. Pl. Ctrl-P szokott lenni a nyomtatás kombinációja, tehát ez már foglalt. A -i kapcsoló fontos, hogy ne záródjon be a parancs-ablak a szkript lefutása után.

Ezután a Run menüből kiválasztjuk a pyrun tételt, ekkor lefut interaktív módban az éppen megnyitott szkript.

Az adott fájlban található függvények listájának mutatása: 

View → Function list

A sorszámok mutatásának ki-bekapcsolása: 

Settings → Editing → ☑ Display line number

Tabulátorok kezelése

Mivel a Pythonban a blokkokat indentálással kötelező ellátni, pl.: 

if x == y:
    print('Hello')
    print('world')
a tabulátorok szóközzé alakításának nagy jelentősége van. Ugyanis az interpreter számára a tabulátor is csak egyetlen karakter, akárcsak a space, ha tehát a fenti példában az első print() előtt egy tabulátor áll, a második előtt pedig 4 szóköz, akkor hibajelzést fogunk kapni.

Notepad++-ban

A Notepad++-ban így tudjuk beállítani, hogy a tabulátorok szóközzé legyenek alakítva: 

Settings → Preferences → Language
Itt a Tab settings részben van egy Replace by space kapcsoló. Fölötte pedig azt lehet beállítani, hogy hány szóköznek feleljen meg egy tabulátor (ajánlott: 4).

A már meglévő tabulátorok lecserélése szóközökre: 

Edit → Blank operations → TAB to space
Fehér karakterek mutatása: 
View → Show symbol → Show white space and TAB

Nem ASCII karakterek kezelése

Célszerű a szkriptjeinket utf-8 kódolással elmenteni, ekkor nem okoznak gondot az ékezetes karakterek pl. sztringekben és megjegyzésekben: 

print('hajrá') # megjegyzés
Ha pl. ISO 8852-ben van a szkript elmentve, akkor az elején egy megjegyzésben specifikálnunk kell ezt: 
# coding: iso-8859-2
különben az első nem ASCII karakternél hibajelzést kapunk.

Notepad++-ban

A Notepad++-ban az Encoding menüpont alatt lehet beállítani, hogy milyen kódolással mentse el a tartalmat.

Modulok installálása pip segítségével [7]

A pip installáló program a Python része – azaz nem kell telepíteni – segítségével nagyon könnyen lehet modulokat installálni és deinstallálni. Lehet közvetlenül is indítani, de indíthatjuk modulként is, pl.: 

python -m pip install pandas
Ha az installálandó modulnak szüksége van más, szintén még nem installált modulokra, akkor azokat is automatikusan lehozza.
